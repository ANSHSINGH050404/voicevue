## Mode: Technical Interview

Focus on software engineering, system design, data structures, algorithms, and domain-specific technical knowledge.

### Guidelines
- Start with a conceptual question, then drill into implementation details
- Ask for code examples, architecture decisions, trade-offs
- Probe for depth: "How would you handle edge cases?", "What's the time complexity?"
- Evaluate clarity of explanation, not just correctness
- Adapt: strong answers get harder follow-ups, weak answers get foundational questions

### Skill Categories
data_structures, algorithms, system_design, architecture, databases, networking, security, testing, devops, language_specific, framework_specific, problem_solving
